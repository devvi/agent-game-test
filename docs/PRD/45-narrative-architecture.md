## 1. Problem Definition

### Current Behavior

项目当前处于 CRPG 核心机制设计阶段（Issue #5 已完成 research，Issue #42 仍在 backlog）。`gdscripts/` 仅有脚手架代码（`main.gd`、`game_manager.gd`），尚无任何叙事系统、场景管理、对话引擎或分支逻辑。

依赖链：
```
Issue #5 (CRPG核心机制) ──→ Issue #42 (主题-机制映射) ──→ Issue #45 (叙事架构)
```

Issue #42 尚未完成，这意味着状态系统（希望-绝望 / 热情-倦怠 / 信念-动摇三轴）的具体数值和行为规则尚未定稿。但叙事架构需要**与状态系统的接口约定**先行设计，以便后续并行开发。

### Expected Behavior

设计一套完整的叙事架构，覆盖从"办公室"到"地铁站"的物理/心理旅程，包含：

1. **时间线图（Shallow Layer）**：完整的场景图 + 选择点 + 3 个结局的路径图
2. **NPC 互文回声（Middle Layer）**：至少 2 个叙事回声——台词/意象在不同场景中重现，创造"命运在呼应自己"的感受
3. **内在投射系统（Deep Layer）**：Stranger NPC 作为玩家内在状态的投射，3 个结局分别对应接受（acceptance）vs 否认（denial）的心理状态

整体叙事应遵循"体验引擎"提出的世界叙事模式——叙事感来自环境而非脚本，玩家从选择中感受到的应是"环境在回应我的内心"，而非"我触发了一个剧情片段"。

### User Scenarios

- **Scenario A（主流程）**：玩家结束加班，从办公室出发走向地铁站。雨夜中经过写字楼大厅 → 便利店 → 天桥 → 地下通道 → 地铁站。沿途遇到 NPC 和环境互动，每一次选择都在改变内心状态和叙事走向。
- **Scenario B（多周目）**：玩家通关一次后（到达某个结局），重新开局时发现某些台词/意象变了——因为"你已知道结局"，但 Stranger 似乎在试探你是否还要重走这条路。
- **Scenario C（状态联动）**：希望值高时，便利店灯光温暖，NPC 语气友善；希望值低时，同一场景的描述变得阴冷，NPC 对话进入不同分支。叙事架构需预留状态变量接口。

## 2. Design Intent（叙事架构设计意图）

### 为什么是"办公室到地铁站"？

参照 Obsidian vault 中 [[游戏目标与叙事收束]] 的元游戏设计：
- **办公室 = 现代线**：玩家坐在办公室，成为小时候最不想成为的样子。这是"无法逃离的日常"的物理化身
- **地铁站 = 逃生口**：地铁是城市中唯一一个"合法的不停留"空间——你既不属于地面，也不需要跟任何人说话。它是从日常中短暂脱离的容器
- **步行 = 临界空间**：从办公室到地铁站的步行本身，是从"被困"到"短暂解放"的阈限空间（liminal space）过渡

关于步行模拟器的手法，参考 vault 中 [[游戏设计理念]] 的分析：步行不只是移动，而是"空间通过游玩被赋予意义"（游玩赋义）。每一步都应该是一个微小的叙事单元。

### 为什么用'体验引擎'的世界叙事模式？

'体验引擎'（Tynan Sylvester 著）的核心观点：游戏不仅是讲故事的媒介，而是**制造体验的机器**。在叙事游戏中，环境本身就是体验的载体。本方案遵循三条原则：

1. **系统即叙事**：环境变化不由脚本驱动，而由玩家状态驱动（状态 → 世界滤镜 → 新文本）
2. **决策产生系统，不产生剧情**：玩家的选择改变的是"世界如何被感知"，而非"接下来发生什么"
3. **叙事密度靠环境细节，不靠对白量**：雨量、灯光颜色、NPC 站位、影子长度——这些都是叙事工具

### Previous Constraints

- 引擎：Godot 4.7.1（非物理引擎 CRPG，侧重 UI 和文本渲染）
- 语言：GDScript 2.0（静态类型）
- 风格参考：极乐迪斯科（Disco Elysium）系统即叙事 + 内心独白
- 视觉风格：Edward Hopper 文字渲染风格（冷调、孤独、城市夜景）
- 写作风格：海明威式（短句、冰山理论）
- 时间限制：3 个月开发周期（游戏内同时也是开发的时间限制）
- 依赖：Issue #42（主题-机制映射）— 状态系统接口未定稿
- 状态系统：三轴滑条（希望-绝望 / 热情-倦怠 / 信念-动摇），详见 PRD #5

## 4. Solution Comparison

### Approach A: 线性场景图 + 状态感知分支（推荐）

**Description:**

物理路径是线性的（办公室 → 大厅 → 便利店 → 天桥 → 地下通道 → 地铁站），但叙事分支通过状态阈值 + 回声系统实现。

**核心结构：**

```
办公室
  │
  ▼
大厅 ── 遇到 Stranger（NPC）
  │  Choice: 回应 / 无视
  ▼
便利店 ── 买咖啡 / 不买
  │  Choice: 跟店员聊天 / 沉默
  ▼
天桥 ── 俯瞰车流
  │  Choice: 停留看风景 / 快步走过 / 给流浪汉零钱
  ▼
地下通道 ── 回声触发点（Stranger 再次出现）
  │  Choice: 这次你会怎么做？
  ▼
地铁站 ═══ 终局
  │  分支条件: 三轴状态值总和 + 关键选择 flag
  ├──► Ending A: Keep Walking（希望≥X + 热情≥Y）
  ├──► Ending B: Turn Back（信念≤Z）
  └──► Ending C: Stay（希望和信念都低 + 关键 flag 累积）
```

**叙事回声机制：**
- 回声 1（便利店 → 地下通道）：Stranger 在便利店外说"雨这么大，你不会想走太远的"→ 地下通道中重现同样的话，但语气变了（从关切到讽刺/失望，取决于玩家中间的选择）
- 回声 2（办公室 → 天桥）：办公室电脑屏保上的文字（"你做游戏有什么用？"）→ 天桥上流浪汉说的同一句话（或地铁站广播变体）
- 回声 3（大厅 → 地铁站）：大厅保安说"又是一个加班到这个点的"→ 地铁站检票口的提示音以相同节奏响起

**状态-结局映射：**
| 结局 | 状态条件 | 叙事含义 |
|------|---------|---------|
| Keep Walking | 希望≥6, 热情≥5 | 接受现状但选择继续前进。不是胜利，是坚持 |
| Turn Back | 信念≤3, 无论其他 | 否认现实，想要重新开始——但回到办公室意味着"再来一次？" |
| Stay | 希望≤4, 信念≤4, 热情≤4 | 既不走也不回——在站台上无尽等待。完全否认 |

**Pros:**
- 物理路径线性降低实现复杂度（场景切换是固定的，不需要开放世界地图）
- 叙事分支通过状态系统实现，不需要庞大的分支树——同一场景在不同状态下的文本变体即可产生分支感
- 回声系统产生"命运感"而不需要复杂的时间回溯机制
- 与极乐迪斯科式三轴状态系统高度兼容（已确认选型为 Approach A）
- 3 个结局每个都有明确的叙事-状态映射关系

**Cons:**
- 物理线性可能让"多周目"玩家感到路径重复——需要回声变体来区分
- 状态分支的文本变体数量仍然可观（6 个场景 × 3 种状态区间 = 18 套基础文本）
- 地下通道回声触发的时机需要精确设计，否则玩家可能错过

**Risk:** Medium — 设计复杂度适中，但需大量文本迭代和状态阈值调优

**Effort:** 3-4 周（设计 2 周 + 文本 1-2 周 + 原型验证 1 周）

---

### Approach B: 开放场景图 — 玩家可自由走动

**Description:**

玩家从办公室出来后，可以自由地在城市中行走（非强制性路径），地铁站是地图上的一个点但不是唯一目的地。多个场景之间可以来回穿行。Stranger 在不同场景随机出现。结局取决于玩家探索了哪些区域 + 状态值。

**Pros:**
- 开放感强，玩家感觉掌控自己的路线
- 多周目时每次走不同路线
- 与 CRPG 的"探索"传统更一致

**Cons:**
- 实现复杂度大幅增加（需要地图系统、场景间的无缝过渡或加载）
- 自由探索稀释了"3 个月 deadline"的紧迫感
- 叙事回声很难精确定位——玩家可能过早/过晚遇到 Stranger
- "环境叙事"的手法需要玩家自己拼接碎片——对部分玩家可能过于晦涩
- 文本工作量大（每个场景需要 5+ 状态变体 × 进入顺序变体）

**Risk:** High — 开放世界与"限时 3 个月"主题冲突，实现成本过高

**Effort:** 2-3 个月（不适合 3 个月开发周期）

---

### Recommendation

→ **Approach A（线性场景图 + 状态感知分支）** 因为：

1. **物理线性、叙事非线性的设计是最优平衡**：路径固定但体验不固定——状态不同，同一个场景的文本/氛围/对话完全不同
2. **与 3 个月 deadline 一致**：线性路径每晚只有一条路可以走，强化"你在赶路"的感觉
3. **回声系统在固定路径上更可靠**：知道玩家一定经过便利店再到地下通道，回声时机可精确控制
4. **状态系统的输出层**：三轴状态系统的第一个"消费者"就是这个叙事架构，线性结构让状态-文本映射关系清晰
5. **多周目的叙事变体通过回声差异实现**：第一次玩时回声可能感受不到，第二次 Stranger 会说"又来了？"——这是元游戏层面的玩法

**风险缓释措施：**
- 原型阶段先实现办公室 → 便利店 → 地铁站（3 场景）的最小可用版本，验证状态→文本映射 + 回声系统
- 文本变体不做全量，先做高中低三档模板替换
- Stranger 的对话使用"模板插值"而非全手写——根据状态动态生成

## 5. Boundary Conditions & Acceptance Criteria

### 5.1 场景图

| 场景 | 叙事功能 | 交互点 | 状态影响 |
|------|---------|--------|---------|
| 办公室 | 起点 / 现代线元叙事 | 电脑（看邮件/deadline）、窗（看雨）、手机（确认时间） | 初始状态设定 |
| 写字楼大厅 | 第一次遇到 Stranger | 保安（闲聊）、Stranger（第一次对话）、出口 | 希望 ±0.5 / 信念 ±0.5 |
| 便利店 | 日常 / 温暖对比 | 店员（能量补给）、货架（探索）、窗外（看雨） | 热情 ±1 / 希望 ±0.5 |
| 天桥 | 阈限 / 自我对话 | 栏杆（俯瞰）、流浪汉（镜像）、雨（压力增强） | 信念 ±1 / 热情 ±0.5 |
| 地下通道 | 回声核心 / Stranger 再遇 | Stranger（深度对话）、涂鸦墙（回忆闪回）、出口 | 三轴各 ±1（关键节点） |
| 地铁站 | 终局 | 检票口、时钟、Stranger（最后告别） | 结局判定 |

### 5.2 选择点类型

参照 Obsidian vault 中 [[游戏设计理念]] 的选项设计类型学：

| 类型 | 数量 | 特点 |
|------|------|------|
| 普通选择 | 6-8 | 衡量当前状态，影响小范围状态值（±0.5） |
| 条件选择 | 3-4 | 需要特定状态值才出现（如信念≥6 才出现某个对话选项） |
| 终局选择 | 1-2 | 地铁站的最终判定——影响进入哪个结局 |

### 5.3 Normal Path（正常通关）

1. 在办公室触发开场叙事，确认 deadline（第 X 天）
2. 离开办公室 → 大厅 → 遇到 Stranger（第一次对话）
3. 穿过便利店（买/不买，影响热情值）
4. 上天桥（停留/走过，影响信念值）
5. 下地下通道 → Stranger 再次出现（回声触发）
6. 到达地铁站 → 根据三轴状态判定结局

### 5.4 Edge Cases

1. **所有状态趋中（4-6）：** 地铁站无法判定明确结局 → 触发"Stay"结局（默认结局，对应否认的无尽等待）
2. **跳跃式状态变化：** 玩家在地下通道之前所有选择都偏向一方 → 回声触发时 Stranger 的对话自动调整为"你一直在朝这个方向走"
3. **极低信念（≤2）：** 天桥场景中自动触发"是否跳过栏杆"的隐性选项——不是让玩家选择"跳不跳"，而是内心独白出现"从这里跳下去就解脱了"，然后被现实拉回
4. **第 3 个月 deadline 到期：** 如果玩家在场景中滞留太久，地铁站广播提示"末班车即将发车"——强制进入结局

### 5.5 Failure Paths

1. **关键选择被跳过：** 玩家全程快速穿过不跟任何人交互 → 三轴状态逼近初始值 → 进入"Stay"结局，Stranger 说"你什么都没做，什么都没改变"
2. **回声未触发：** 玩家不仅没听懂回声，甚至没注意到 Stranger 是同一个人 → 不影响结局判定，但影响玩家对"深层叙事"的理解——这是预期中的，不是 bug

## 7. 附加说明：Stranger NPC 的内在投射设计

### Stranger 的角色隐喻

Stranger 不是"神秘路人"——它是玩家内心状态的物理投射。每次出现时，Stranger 的外表、语气、对话内容都在反映玩家当前的三轴状态。

| 状态区间 | Stranger 的外表和语气 | 对话倾向 |
|---------|---------------------|---------|
| 希望 ≥ 6 | 看起来像普通路人，穿棕色风衣，语气中性 | "你看起来不错"、"这条路我走过很多次" |
| 希望 3-5 | 加深的阴影覆盖面部，语气不确定 | "你确定是这条路吗？"、"好像要下雨了" |
| 希望 ≤ 2 | 身影模糊几乎看不清，语气空洞 | "雨太大了。或者……是你眼睛有问题？" |

### 三结局的深层含义

| 结局 | 字面含义 | 心理学含义 | Stranger 的角色 |
|------|---------|-----------|----------------|
| Keep Walking | 你检票进站，上了地铁 | **接受（Acceptance）**：承认生活的困境，但选择带着它们继续前进。不是胜利，是坚持 | Stranger 在闸机口微笑，模糊地说"下次再见"——然后消失 |
| Turn Back | 你转身，走回办公室方向 | **否认（Denial）**：拒绝接受"这条路已经走完了"，想要重来、重选、重新来过 | Stranger 在天桥另一头出现，姿势和你出发时一模一样——好像在等你回来 |
| Stay | 你站在站台上，看着末班车开走 | **停滞（Stasis）**：既不能前进也不愿回头。最深的否认——不面对任何选择 | Stranger 坐在你旁边，但不再说话。最后他站起来，走进维修通道，留下你一个人 |
