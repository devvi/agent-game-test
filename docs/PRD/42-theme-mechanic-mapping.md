## 1. 问题定义

### 当前状态

Issue #5（CRPG核心机制设计）已定义了三轴滑条系统（希望-绝望 / 热情-倦怠 / 信念-动摇）作为Project YOUTH的基础机制框架，并产出了5个提案机制（对话即检定、世界观滤镜、雨夜压力、3个月时钟、三轴滑条）的初步映射链。

但当前的映射链是**定性且未评分**的——每个机制与主题的绑定关系有描述但缺乏：
1. **结构性映射链**：主题节拍 → 玩家感知 → 机制动作 → 情感输出的完整链路
2. **量化和谐度评分**：无法区分"核心绑定"与"锦上添花"
3. **系统性验证**：没有替代测试和消除测试来验证绑定的必要性

### 预期状态

产出一份结构化的主题-机制映射文档，满足：
1. 至少5条从**主题节拍**到**机制动作**的完整映射链
2. 每条映射链附带**和谐度评分（1-5）**及详细论证
3. 至少2个**替代测试**和2个**消除测试**，验证映射的必要性和排他性

### 用户场景

- **场景A（设计者）:** Game Designer 需要知道"哪些机制绝对不可替换"——在3个月开发周期中，砍掉锦上添花的机制保留核心机制
- **场景B（叙事作者）:** Writer 需要知道"每个文本变体服务于哪个主题节拍"——避免写出与主题无关的冗余内容
- **场景C（系统程序员）:** Programmer 需要知道"机制与机制之间的绑定强度"——决定哪些系统应该紧耦合、哪些可以松耦合
- **频率:** 一次性设计产出，但将作为Issue #5以来所有后续机制决策的参考框架

## 2. 设计意图

### 为什么需要这份文档？

Project YOUTH 的核心命题是"一个人的希望在雨夜中如何被不同的人和经历改变"。这个命题不是通过一个机制实现的，而是通过**一组机制的协同**。但协同意味着有的机制是骨架、有的是血肉、有的是装饰。

当前的设计文档（PRD #5）将所有机制并列呈现，没有区分它们的**主题绑定强度**。当我们说"去掉雨夜压力系统"时，需要具体知道：主题还剩多少？哪个主题节拍彻底消失了？哪个节拍还可以被其他机制承载？

### 为什么现在做？

Issue #2（叙事架构与结局图谱）、Issue #4（状态-世界反馈系统设计）和Issue #6（对话引擎数据模型）都依赖#42的映射结果。如果不先回答"这个机制到底服务哪个主题节拍"，后续的文本创作将缺乏优先级依据。

### 参考资料与约束

- **Project YOUTH 原始笔记**（来自Obsidian vault `wiki/Project YOUTH.md`）：
  - 游戏即表达媒介——机制本身即叙事语言
  - 主角是转校生"壳"——被复活的人，记忆模糊，核心冲突是"我到底是谁"（身份焦虑）
  - 召唤仪式元叙事——游戏本身就是一种召唤，让死去的同学在游戏内"复生"
  - 青春期特有元素（憋尿、外观焦虑、他人评价）——日常尴尬是身份焦虑的微观体现

- **体验引擎设计模式**（来自Obsidian vault `wiki/体验引擎-patterns.md`）：
  - **模式#10: 决策生产系统** — 通过机制交互自然产生决策，而非脚本化选择点。核心结论：对话不应是"选A或B"，而是"你当前的状态决定了你怎么回应"。
  - **模式#6: 标签化** — 用叙事层赋予抽象机制情感意义。滑条值不应显示为数字，而应显示为内心独白的语气。
  - **模式#5: 世界叙事** — 环境本身就是故事载体。雨夜不是背景，而是叙事媒介。
  - **模式#4: 情绪维持** — 在玩家学会系统之前先用氛围留住他们。

- **PRD #5 的已有机制定义**：
  | 机制 | 核心定义 |
  |------|---------|
  | M1: 对话即状态检定 | 对话中的回应选择受当前状态值影响，而非纯叙事分支 |
  | M2: 希望绝望世界观滤镜 | 状态值改变场景文本的冷暖色调和描述语气 |
  | M3: 雨夜压力系统 | 信念值低 → 雨更大 → 迫使避雨 → 触发内心对话 |
  | M4: 3个月deadline时钟 | 每场对话消耗游戏天数，产生时间紧迫感 |
  | M5: 三轴滑条结构 | 三个独立维度（希望/热情/信念）而非单一数值 |
  | M6: 主动感知凝视 | 玩家主动凝视场景对象，获得当前状态滤镜下的独特独白 |
  | M7: 滤镜切换微交互 | 在对话中短暂切换视角，看到NPC的另一面 |

- **身份焦虑主题来源**（Project YOUTH 核心设定）：
  - "转校生壳"的设定本身就是身份焦虑的具象化：我不确定我是谁
  - 外观系统（衣服品牌、发型）是身份表达的外部化
  - 他人评价即内心状态的外在反馈机制
  - 游戏即召唤仪式的元叙事：我的存在本身就是一个问号

## 3. 映射链设计

### 3.1 主题-机制映射总览

| 映射链 | 主题节拍 | 主机制 | 辅助机制 | 和谐度 |
|--------|---------|--------|---------|-------|
| C1 | 绝望感：行业困境与3个月deadline的压迫 | M4: 3个月时钟 | M3: 雨夜压力 | 5/5 |
| C2 | 希望感：微小善意的积累与自我救赎可能 | M2: 世界观滤镜 | M6: 主动感知凝视 | 5/5 |
| C3 | 身份焦虑：对自己的不确定和他人眼中的自己 | M1: 对话即检定 | M7: 滤镜切换 | 5/5 |
| C4 | 存在证明：被记住与被忘记的恐惧 | M2: 世界观滤镜 | M5: 三轴滑条 | 4/5 |
| C5 | 青春期的对外投射：外观焦虑与自我呈现 | M5: 三轴滑条（信念轴） | M6: 主动感知凝视 | 4/5 |
| C6 | 情感回归：在雨夜中被迫与自我对话 | M3: 雨夜压力 | M1: 对话即检定 | 4/5 |
| C7 | 复杂人格：人的状态不是非黑即白 | M5: 三轴滑条 | M7: 滤镜切换 | 3/5 |

### 3.2 详细映射链

#### 映射链 C1: 绝望感 → 3个月deadline时钟 + 雨夜压力

**主题节拍分析：**
> "3个月做出卖座游戏的绝望感和行业困境"（PRD #5）
> 游戏本身就是一种召唤仪式，有时间限制（Project YOUTH scope定义）

**映射链路：**
```
主题节拍（绝望：deadline不可逃避）
  → 玩家情感（紧迫、焦虑、窒息感）
  → 机制动作 M4（每场对话消耗游戏天数、每周deadline提醒）
  → 情感输出（时间在流逝，我什么都没做成）
  ↑ 叠加
  → 机制动作 M3（信念值越低雨越大，雨中行动消耗更多热情）
  → 情感输出（环境也不放过我，无处可逃）
  → 合成体验（"做游戏是在和时间赛跑，而终点不一定是成功"）
```

**体验引擎决策生产系统映射：**
M4不是显式倒计时，而是一种**隐式资源决策**：每次对话玩家都在问自己"这个对话值不值得花几天时间？"——这正是模式#10"决策生产系统"的应用。玩家不是在阅读剧情，而是在做资源分配决策。

**主题验证：**
去掉此映射 → "3个月"变成背景设定而非可感体验 → 绝望感从"正在经历"退化为"被告知" → 主题完整度↓70%

---

#### 映射链 C2: 希望感 → 世界观滤镜 + 主动感知凝视

**主题节拍分析：**
> "便利店灯光的冷暖变化"（PRD #5 深夜便利店场景）
> "世界怎么呈现，取决于你怎么看"（PRD #5 映射链B）
> 希望不是HP，是决定了世界如何呈现的滤镜（PRD #5）

**映射链路：**
```
主题节拍（希望：即便在最暗的夜也有微光）
  → 玩家情感（渴望温暖、寻找意义）
  → 机制动作 M2（希望值≥7：灯光泛黄，雨声像白噪音；希望值≤3：灯光苍白，雨声像敲打骨头）
  → 情感输出（我的内心状态改变了世界，世界就不再那么冰冷）
  ↑ 增强
  → 机制动作 M6（长按凝视路灯 → 高希望："光暖洋洋的，让人想多待一会儿"；低希望："光太冷了，像是医院走廊"）
  → 情感输出（同一件事，不同的我看到不同的风景）
  → 合成体验（"希望不是幻想，是你看世界的方式"）
```

**体验引擎标签化模式映射：**
M6是对模式#6"标签化"的实践——抽象机制（滑条值变化）通过叙事文本（凝视后的独白）获得情感意义。玩家不是在"加数值"，而是在"重新学习如何看待世界"。

**主题验证：**
去掉此映射 → 希望变成纯数值，无感性反馈 → 玩家无法"感觉到"希望的转变 → 希望主题变成机械化的加减分 → 主题完整度↓80%

---

#### 映射链 C3: 身份焦虑 → 对话即检定 + 滤镜切换

**主题节拍分析：**
> "主角是个转校生'壳'，实则是被复活的人，复活之后记忆模糊，核心包袱：想搞清楚自己是谁"（Project YOUTH）
> "NPC对话不仅是叙事，是一次状态检定"（PRD #5）
> "人的状态是复杂的，拯救了一面的同时可能失去另一面"（PRD #5）

**映射链路：**
```
主题节拍（身份焦虑：我是谁？我是别人眼中的我还是真实的我？）
  → 玩家情感（迷惑、不确定、想要被认可）
  → 机制动作 M1（对话中NPC问"你还在做游戏？"→ 信念≥5时坦然回答，信念<5时只能回避）
  → 情感输出（我无法控制自己怎么回答——状态替我选择了）
  ↑ 反射
  → 机制动作 M7（在对话中短暂切换信念视角看 NPC → 高信念：看到对方的坚定；低信念：看到对方的不安）
  → 情感输出（原来不同状态下的我，看到的世界完全不同）
  → 合成体验（"身份不是你想成为谁，而是你现在的状态决定了你以为自己是谁"）
```

**体验引擎决策生产系统映射：**
M1的决策不是"选A或B"，而是**玩家当前状态自动决定了默认倾向**。玩家当然可以选择反状态行事（希望低却假装乐观），但需要付出代价（倦怠+1）。这创造了模式#10所描述的"由机制自然产生的决策"——玩家不是在走剧情树，而是在管理自我认知。

**主题验证：**
去掉此映射 → 对话变成纯叙事交互，无法体现"状态决定了我的表达" → 身份焦虑变成了背景故事（被告知"你很迷茫"），而非可感体验 → 主题完整度↓85%

---

#### 映射链 C4: 存在证明 → 世界观滤镜 + 三轴滑条

**主题节拍分析：**
> "游戏本身是一种召唤仪式——让死去的同学通过游戏在游戏内'复生'"（Project YOUTH）
> "穿不同的衣服，在意别人的评价"（Project YOUTH）
> "被记住才能存在"（Project YOUTH 元叙事）

**映射链路：**
```
主题节拍（存在证明：在这个世界上留下痕迹，被记住）
  → 玩家情感（恐惧被遗忘、渴望被看见）
  → 机制动作 M2（世界描述的冷暖随状态变化 — 高信念时NPC记住你的名字，低信念时NPC记不住）
  → 情感输出（世界对我的反馈，证明了我是存在的）
  ↑ 基准
  → 机制动作 M5（三个独立维度记录你的每一个选择，形成完整的"你"的画像）
  → 情感输出（每一个微不足道的选择都在塑造'我'）
  → 合成体验（"你做过什么，你就成为什么。被记住的，不是名字，是你选择的方式。"）
```

**体验引擎抽象留白模式映射：**
M2的文本变体不需要写满所有状态组合。利用模式#7"抽象留白"——只在滑条到达极端值（1或10）时给出明显的描述变化，中间值让玩家自己感受。这降低了文本量，同时保留了想象空间。

**主题验证：**
去掉此映射 → 游戏变成"只是讲了一个关于被遗忘的人的故事"，玩家无法亲身感受"世界对我的反馈变弱了"这种存在感侵蚀 → 主题完整度↓60%

---

#### 映射链 C5: 青少年对外投射（外观焦虑/自我呈现） → 信念轴 + 主动凝视

**主题节拍分析：**
> "可以买衣服穿，不同的05年到10年的牌子，比如班尼路，不同的衣服有不同的效果"（Project YOUTH）
> "青春期的少男少女非常在意自己的外观，在意别人的评价，还有发型"（Project YOUTH）
> "憋尿——青春期日常尴尬"（Project YOUTH）

**映射链路：**
```
主题节拍（外观焦虑：我穿什么决定了别人怎么看我，别人怎么看我决定了我怎么看我）
  → 玩家情感（不安、在意、渴望被认可、羞耻感）
  → 机制动作 M5-信念轴（穿不同品牌触发不同的NPC评价 → 评价反馈影响信念值）
  → 情感输出（即使我不愿意承认，别人的看法确实影响了我）
  ↑ 互文
  → 机制动作 M6（凝视镜子中的自己 → 当前信念值决定独白："这身还行" vs "我看起来像个小丑"）
  → 情感输出（我在意的不只是衣服，是"穿这件衣服的我"在别人眼中是谁）
  → 合成体验（"青春期是你第一次意识到，你的存在取决于别人怎么看你"）
```

**体验引擎标签化模式映射：**
衣服选择本身是模式#6"标签化"的载体——班尼路、以纯等品牌不只是视觉外观，每个品牌携带特定的社会标签（阶层、品味、地域）。玩家买衣服不是在换装，是在选择"我想被看作什么样的人"。

**主题验证：**
去掉此映射 → 外观系统变成纯装饰（相当于没有机制作用） → 青少年的"被看见焦虑"从核心体验降级为背景点缀 → 主题完整度↓55%

---

#### 映射链 C6: 被迫内省（情感回归） → 雨夜压力 → 对话即检定

**主题节拍分析：**
> "雨夜作为环境本身就是一种压力系统"（PRD #5）
> "信念值低 → 雨更大 → 被迫避雨 → 触发被迫停下的内心对话"（PRD #5）
> "无处可逃"（PRD #5）

**映射链路：**
```
主题节拍（被迫内省：当外部世界不再允许你逃避，你必须面对自己）
  → 玩家情感（无奈、被迫、不情愿但不得不面对）
  → 机制动作 M3（信念≤3时暴雨触发→玩家必须在雨棚下停留45s→雨棚内只能触发"内心独白"交互，无其他操作）
  → 情感输出（我不想面对自己，但雨太大走不了）
  ↑ 强迫
  → 机制动作 M1（雨棚停留触发强制状态检定——"你有多久没问过自己为什么还在做这个了？"）
  → 情感输出（无法逃避的问题，就是最核心的问题）
  → 合成体验（"雨夜不让你跑，逼你停下来想一想。你躲得了NPC，躲不了自己。"）
```

**体验引擎情绪维持模式映射：**
M3的避雨场景是对模式#4"情绪维持"的逆向应用——不是用美好氛围留住玩家，而是用压抑氛围迫使玩家在一段时间内只能做"面对自己"这一件事。恶劣环境反而成为最有情感密度的空间。

**主题验证：**
去掉此映射 → 玩家可以一直"做事情"，永远不需要面对内心独白 → "被迫内省"这个核心叙事节拍完全消失 → 主题完整度↓75%

---

#### 映射链 C7: 复杂人格（补充映射——和谐度3/5，锦上添花） → 三轴滑条 + 滤镜切换

**主题节拍分析：**
> "人的状态是复杂的，一个选择可能提升希望但降低信念"（PRD #5）
> "状态不能对角翻转，每条轴朝一个方向移动后需要触发事件才能折返"（PRD #5）

**映射链路：**
```
主题节拍（复杂人格：你不是只有一种情绪，你同时拥有希望和恐惧）
  → 玩家情感（矛盾、理解、自我审视）
  → 机制动作 M5（三轴独立变化：帮NPC → 希望+1但热情-0.5，熬夜改bug → 热情-2但信念+0.5）
  → 情感输出（我做的每一件事都有代价，没有完美的选择）
  ↑ 增强
  → 机制动作 M7（滤镜切换微交互：用高信念看同一段回忆 → 看到坚持的意义；用低信念看 → 看到徒劳）
  → 情感输出（记忆本身是中性的，是我的状态给它着色）
  → 合成体验（"人不是单纯的'好人'或'坏人'，你是所有选择的总和"）
```

**体验引擎技能范围模型映射：**
三轴滑条（M5）对应模式#11"自我重塑设计"的操作→战术→心理进阶：最初玩家只感知单轴变化（操作层），逐渐理解轴间互动（战术层），最终洞察"没有完美状态"的设计哲学（心理层）。

**主题验证：**
去掉此映射 → 游戏依然成立（单轴"希望值"也能讲完故事），但失去了"人格深度"的文学性 → 主题完整度↓35%

## 4. 和谐度评分体系

### 4.1 评分方法

每对主题-机制映射从以下5个维度评分，最终取均值（四舍五入）：

| 维度 | 权重 | 说明 |
|------|------|------|
| T. 主题绑定强度 | 1.0 | 此机制是否是表达该主题节拍**最有效**的手段？去掉此机制后，该主题是否仍能传达？ |
| E. 情感精准度 | 1.0 | 此机制产生的情感与该主题节拍的目标情感是否**一致**？（而非产生相反或不相关情感） |
| U. 玩家感知度 | 1.0 | 玩家是否能**明显感知**到这个机制与主题的关联？还是需要看设计文档才明白？ |
| I. 不可替代性 | 1.0 | 是否有其他机制能达到同样的主题表达效果？越低越不可替代。 |
| C. 一致性 | 1.0 | 此机制与其他机制的和谐度——是否与其他映射链冲突或矛盾？ |

### 4.2 评分结果

| 映射链 | T(主题绑定) | E(情感精准) | U(玩家感知) | I(不可替代) | C(一致性) | **总分** | 定性 |
|--------|:---------:|:---------:|:---------:|:---------:|:---------:|:-------:|:----:|
| C1: 绝望→时钟+雨夜 | 5 | 5 | 5 | 5 | 5 | **5/5** | 🔵 核心骨架 |
| C2: 希望→滤镜+凝视 | 5 | 5 | 5 | 4 | 5 | **5/5** | 🔵 核心骨架 |
| C3: 身份→对话检定+滤镜切换 | 5 | 5 | 4 | 5 | 5 | **5/5** | 🔵 核心骨架 |
| C4: 存在→滤镜+三轴 | 4 | 4 | 4 | 4 | 5 | **4/5** | 🟢 重要支撑 |
| C5: 外观→信念轴+凝视 | 4 | 4 | 4 | 4 | 4 | **4/5** | 🟢 重要支撑 |
| C6: 内省→雨夜+检定 | 5 | 5 | 4 | 3 | 4 | **4/5** | 🟢 重要支撑 |
| C7: 复杂→三轴+切换 | 3 | 3 | 3 | 3 | 4 | **3/5** | 🟡 锦上添花 |

### 4.3 评分细则（部分展开）

**C1（绝望→时钟+雨夜）评分 5/5 理由：**
- T=5: "3个月deadline"是绝望的物理化身，没有时钟就没有"时间在流逝"的身体感
- E=5: 倒计时产生焦虑和紧迫感——这正是"绝望"的核心情感
- U=5: 玩家直接在UI和时间感知中感受到deadline，不需要解释
- I=5: 没有任何其他机制可以替代"时间流逝"的身体感——滤镜不行，对话检定也不行
- C=5: 时钟消耗天数本身与对话系统绑定（每场对话消耗时间），自然融合

**C3（身份焦虑→对话即检定+滤镜切换）评分 5/5 理由：**
- T=5: 身份焦虑的核心是"我在不同状态下说不同的话、看到不同的世界"——对话即检定直接就是这种体验的具象化
- E=5: 对话检定产生"我无法控制自己"的不确定感——这正是身份焦虑的情感核心
- U=5: 玩家在第一次对话中就会意识到"我选择了不是我想选的选项"——无需任何教程
- I=5: 无法被替代——纯叙事树或纯flag系统都无法模拟"状态自动影响选择"的体验
- C=5: 与M2（滤镜）和M5（三轴）自然配合——对话结果影响滑条，滑条影响后续对话

**C7（复杂人格→三轴+切换）评分 3/5 理由：**
- T=3: "人的复杂性"更偏向文学性的升华，而非玩家可感的主题节拍——大多数玩家不会主动追求"理解人格复杂性"
- E=3: "复杂人格"是理性认知而非强烈情感——玩家的情感更多来自单轴的变化而非多轴的互动
- U=3: 三轴互动需要玩家至少玩过2个场景才能感知——新人玩家第一印象只是"有几个数值在变"
- I=3: 单轴系统（只有希望值）也能讲完整的故事——复杂人格是加分项不是必需品
- C=4: 与其他机制没有矛盾，但也没有强绑定关系

## 5. 边界条件与验收标准

### 5.1 替代测试（Substitution Test）

> 替代测试：将一个机制放到完全不同的游戏中，看是否仍然合理。目的是验证该机制是否**排他性地**绑定于本主题。

#### 替代测试 1: 将 M4（3个月deadline时钟）放入《星露谷物语》

| 维度 | 评估 |
|------|------|
| **结果** | ✓ 完全合理 |
| **为什么？** | 时间管理是《星露谷物语》的核心机制——每天24小时、季节性事件、年度评分。M4的本质是"时间资源管理"，在种田游戏中自然成立。 |
| **结论** | M4（时钟）不是Project YOUTH主题**独占**的机制。它绑定的是**通用焦虑**（时间不够），而非**特定焦虑**（做不出游戏的绝望）。 |
| **对设计的启示** | M4虽然和谐度5/5，但并不是"只有Project YOUTH才能用的机制"。它的主题绑定源于**上下文**（deadline是3个月的开发周期+行业困境），而非机制本身。因此在实现时，M4的UI和叙事上下文**必须**与行业困境深度绑定（deadline提醒文本必须是关于"做不出游戏"的焦虑，而非通用倒计时）。 |

#### 替代测试 2: 将 M1（对话即检定）放入《女神异闻录5》

| 维度 | 评估 |
|------|------|
| **结果** | ✗ 基本不合理 |
| **为什么？** | P5的对话系统是选择驱动的——你的选择提升某个Coop等级，获得特定能力。P5的机制核心是"主动选择 → 获得奖励"。而M1的核心是"状态驱动选择 → 自我认知变化"。M1中玩家**不能自由选择**——状态值低时某些选项根本不出现。这与P5的"自由选择 = 自由奖励"设计哲学冲突。 |
| **结论** | M1（对话即检定）是高度主题专属的机制。它只有在"内心状态决定表达"的叙事CRPG中才能发挥作用。在传统JRPG中，M1会被玩家感知为"系统替我做决定"的挫败感。 |
| **对设计的启示** | M1不可被替代（I=5的实证理由）——它是Project YOUTH区别于传统RPG的核心识别点。实现时必须**明确教育玩家**：不是"系统不让我选"，而是"现在的我不可能那么说"。 |

---

### 5.2 消除测试（Elimination Test）

> 消除测试：去除一个机制后，评估主题节拍是否仍然完整。目的是验证该机制对主题表达的**必要性**。

#### 消除测试 1: 去除 M3（雨夜压力系统）

**去除后状态：**
- 雨变成纯背景视觉（不再随信念值变化）
- 玩家可以在雨中无限行走，无事发生
- 没有避雨触发内心对话的机制

**生存的主题节拍：**
- C1（绝望→时钟）：仍然存在 — deadline时钟独立运作
- C2（希望→滤镜）：仍然存在 — 文字滤镜不需要雨
- C3（身份→检定）：仍然存在 — 对话检定独立运作

**消失或减弱的主题节拍：**
- **C6（被迫内省）：完全消失** — 唯一强制玩家"停下来面对自己"的机制没有了。玩家可以永远"做事情"。
- **环境即叙事的沉浸感：显著减弱** — 雨从"角色"降级为"背景板"

**结论：** M3（雨夜压力）**不是**对整体主题表达必要的机制（因为C1/C2/C3仍然存活），但**是"C6被迫内省"这个特定节拍的必要条件**。如果"C6被迫内省"是核心设计目标（项目定义中"游戏即表达"所需），则M3必须保留。

**行动建议：** 保留M3但降低其实现优先级。可以在第一次迭代中只做"信念值影响雨量"（M3的50%），避雨强制对话放在第二次迭代。

#### 消除测试 2: 去除 M5（三轴滑条结构），替换为单值系统

**去除后状态：**
- 希望-绝望 / 热情-倦怠 / 信念-动摇 三轴 → 压缩为单轴"精神状态"（1-10）
- 所有状态检定变为单值区间匹配

**生存的主题节拍：**
- C1（绝望→时钟）：仍然存在 — 时钟不依赖多轴
- C2（希望→滤镜）：部分存在 — 但只有一种"颜色"变化（黑白冷暖），失去了"热情让世界嘈杂、信念让世界清晰"的多维感知
- C3（身份→检定）：显著减弱 — "我在不同维度上的不同位置"这种复杂性消失
- C4（存在→滤镜）：减弱 — 世界反馈只有一维变化

**消失或减弱的主题节拍：**
- **C5（外观多元）：显著减弱** — 衣服品牌不再是"你看自己的方式"，变成"加1点希望"的简单道具
- **C7（复杂人格）：完全消失** — 单轴无法产生"希望上升但信念下降"的矛盾体验

**结论：** M5（三轴）**不是**对基础叙事完整性必要的机制（单轴也能讲完"一个绝望的人重拾希望"的故事），但**是"身份复杂性和青春期多维焦虑"主题的必要载体**。

**行动建议：** 保留三轴但可将"热情轴"的文本变体数量减到最少（因为C7和谐度仅3/5），优先保证"希望轴"和"信念轴"的丰富度。这与PRD #5的"原型阶段先做2组滑条"建议一致。

## 7. 连续上下文

> *本节是activeForm对下一agent（plan → implement）的交接上下文。*

Project YOUTH 的主题-机制映射已完成结构化定义。核心结论：

**三个不可替代的核心骨架机制**（来自替代测试2和消除测试的实证验证）：
