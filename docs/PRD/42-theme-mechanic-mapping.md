# Research: 主题-机制映射链设计

> Parent Issue: #42
> Agent: game-research-agent
> Date: 2026-07-22

---

## 1. Problem Definition

### Current Behavior

项目当前处于 CRPG 核心机制设计阶段（Issue #5 已完成 PRD，#6 脚手架已就位）。`gdscripts/` 中包含 `main.gd`（标准入口）和 `game_manager.gd`（Autoload 管理器骨架），`scenes/` 中只有 `main.tscn` 一个场景。尚无运行时状态系统、对话引擎或主题-机制映射层的任何代码。

主题层面，Issue #5 的 PRD 已定义了三组状态滑条（希望-绝望 / 热情-倦怠 / 信念-动摇）以及五大机制（对话即检定 / 世界观滤镜 / 雨夜压力 / 3 个月时钟 / 三轴滑条），但这些机制与主题之间的**映射链**尚未形成独立、可验证的设计文档。

### Expected Behavior

产出一份独立的**主题-机制映射链文档**，清晰回答以下问题：

1. 每个主题 beat（绝望感、希望感、身份焦虑）被哪些机制承载？
2. 每个机制如何具体实现主题表达（机制动作 → 玩家行为 → 主题呈现的完整链路）？
3. 每个映射对的**和谐度评分**（1-5 分 + 理由）是多少？
4. 如果替换某个机制到不同主题的游戏，或完全去掉某个机制，主题是否还能存活？

### User Scenarios

- **Scenario A（设计者视角）:** 设计者需要理解「希望感」到底由哪个系统负责、如何触发、玩家如何感知。映射链提供了从"叙事意图"到"GDScript 代码"的桥梁。
- **Scenario B（验证者视角）:** Reviewer 需要判断某个机制是否与主题强耦合。替换/消除测试提供了客观判断依据。
- **Scenario C（实现者视角）:** 开发者需要知道每个映射对在 Godot 4.7 中的实现优先级和依赖关系。

## 2. Design Intent

### Why Does Current Behavior Exist?

项目目前完成了基础设施（#1 Hello World, #6 脚手架）和宏观机制设计（#5 CRPG 核心机制），但尚未进入"主题 ↔ 机制"层面的细化映射。Issue #42 的目标是填补这个中间层——设计文档（#5 → docs/PRD/）与后续实现（#10 GameState 系统等）之间的分析桥梁。

### Why Change Now?

- 宏观机制列表已定（五大机制），但各机制与主题的**绑定强度**未量化
- 后续 Issue（#2 叙事架构、#4 状态-世界反馈）需要明确的映射链来指导实现
- 替换/消除测试应在代码实现之前完成，避免在运行时发现设计缺陷
- 「Project YOUTH」笔记和「体验引擎」方法论需要具体落地到 Godot 项目中

### Previous Constraints

| 约束 | 内容 |
|------|------|
| 引擎 | Godot 4.7.1 / GDScript 2.0（静态类型） |
| 主题集 | 三个核心主题：绝望、希望、身份 |
| 参考风格 | 极乐迪斯科（Disco Elysium）— 系统即叙事 |
| 写作风格 | 海明威式（短句、冰山理论） |
| 开发周期 | 3 个月（游戏内 90 分钟） |
| 已有机制 | 五大机制（对话即检定、世界观滤镜、雨夜压力、3 月时钟、三轴滑条） |

## 4. Solution Comparison

### Approach A: 定向映射（Theme-Driven Mapping）

**Description:**

以**主题**为中心，先拆解三个核心主题的子成分，再为每个子成分寻找最合适的机制载体。

| 主题 | 子成分 | 最佳机制载体 |
|------|--------|-------------|
| 绝望 | 无力感 / 被抛弃 / 自我否定 | 世界观滤镜 + 对话检定 |
| 希望 | 微光 / 他人善意 / 自我肯定 | 对话检定 + 三轴滑条 |
| 身份焦虑 | 我是谁 / 我属于这里吗 / 我在做什么 | 对话检定 + 三轴滑条 |

**Pros:**
- 主题→机制链路清晰，设计可追溯
- 每个子成分都有明确的责任机制
- 便于后续添加新主题/子成分时做增量映射

**Cons:**
- 可能遗漏机制的"第二主题影响"（一个机制可能承载多个主题）
- 需要精细的子成分分解

**Risk:** Low — 结构清晰，PRD #5 已有部分基础

**Effort:** 1 天设计 + 1 天文档

### Approach B: 机制回溯（Mechanic-Driven Mapping）

**Description:**

以**机制**为中心，对五大机制分别问"这个机制表达了哪些主题？"

| 机制 | 主要表达主题 | 次要表达主题 |
|------|-------------|-------------|
| 对话即检定 | 身份焦虑 + 希望 | 绝望 |
| 世界观滤镜 | 绝望 + 希望 | 身份焦虑 |
| 雨夜压力 | 绝望 | 身份焦虑 |
| 3 月时钟 | 绝望 | — |
| 三轴滑条 | 身份焦虑 | 希望 + 绝望 |

**Pros:**
- 保证每个机制都被映射到主题
- 不会遗漏机制的"第二主题影响"
- 适合用于验证 Approach A 的结果

**Cons:**
- 主题视角不够强，可能产生多条不重要的映射
- 输出结构不如 Approach A 直观

**Risk:** Low — 简单直接

**Effort:** 半天

### Approach C: 双向映射矩阵（Bidirectional Mapping Matrix）

**Description:**

结合 Approach A 和 B，构建一个二维矩阵：

|  | 对话即检定 | 世界观滤镜 | 雨夜压力 | 3 月时钟 | 三轴滑条 |
|--|-----------|-----------|---------|---------|---------|
| 无力感 | 5 | 5 | 4 | 3 | 4 |
| 被抛弃 | 3 | 5 | 5 | 1 | 2 |
| 自我否定 | 5 | 3 | 2 | 1 | 5 |
| 他人善意 | 5 | 3 | 0 | 0 | 2 |
| 自我肯定 | 5 | 1 | 0 | 0 | 4 |
| "我是谁" | 5 | 2 | 1 | 1 | 5 |
| "我属于这里吗" | 4 | 3 | 3 | 0 | 4 |
| "我在做什么" | 3 | 2 | 2 | 5 | 3 |

**Pros:**
- 最全面的映射覆盖
- 矩阵形式直观显示强弱映射
- 可直接从中提取映射链（AC1）、和谐度评分（AC2）

**Cons:**
- 输出文档较长
- 需要更细致的子成分分解
- 可能过度工程化

**Risk:** Low-Medium — 信息量最大但文档最重

**Effort:** 2 天设计 + 1 天文档

### Recommendation

→ **Approach C（双向映射矩阵）**，但以 Approach A 的叙事体段落补充矩阵的"温度"。理由：

1. AC1 要求至少 5 条映射链 — 矩阵天然提供 24+ 个映射对
2. AC2 要求和谐度评分 — 矩阵中的数字本身就是评分
3. AC3 要求替换/消除测试 — 矩阵的"0"分项天然标识消除候选，跨主题替换需额外分析
4. PRD #5 已有类似评分表（5.8 节）和替换测试（5.7 节），本 PRD 将其从"宏观机制评分"升级为"子成分级映射矩阵"

## 5. Boundary Conditions & Acceptance Criteria

### 5.1 映射链声明（AC1 — 至少 5 条）

**映射链 1：无力感 → 世界观滤镜**
- **链路：** 玩家内心无力感上升 → 世界观滤镜切换到"冷/灰色调" → 环境文字出现"残酷变体"（便利店的灯"刺眼又苍白"） → 玩家感知到"世界变冷了" → 强化无力感认知
- **Godot 实现：** `GameManager` 广播 `state_changed` 信号 → 场景文本模块根据 `hope` 值切换描述模板
- **反向验证：** 去掉滤镜后，无力感仅存在于对话中，失去"世界在反映我"的沉浸感

**映射链 2：他人善意 → 对话即检定**
- **链路：** NPC 触发对话 → 玩家当前状态值（希望/信念）决定对话分支 → 高希望时 NPC 语气温和 → 玩家感受到"还有人关心我" → 希望上升
- **Godot 实现：** `dialogue_engine.gd` 根据 `hope >= 7` 或 `conviction >= 5` 选择对话分支 ID → 文本资源池加载对应变体
- **反向验证：** 去掉检定后，他人善意变成固定台词，玩家的状态对叙事无影响

**映射链 3：自我否定 → 三轴滑条 + 对话即检定**
- **链路：** NPC 问"你为什么还在做这个？" → 对话检定触发 → 玩家选择 A（"因为我想做"）→ 信念 +1，希望 +0.5 → 内心独白文本切换为坚定风格
- **Godot 实现：** 选择 ID → `state_system.gd.apply_choice()` → 修改信念/希望值 → `state_changed` 信号 → 内心独白模块切换
- **反向验证：** 去掉三轴滑条后，"自我否定"变成二元 flag（有/无），失去了"程度感"

**映射链 4：被抛弃感 → 雨夜压力**
- **链路：** 信念值下降 → 雨量增大 → 被迫找地方避雨 → 避雨期间触发回忆思考 → 玩家感知到"连天气都在和我作对"
- **Godot 实现：** 雨量 = f(信念值, 游戏内时间)，每 30s 检查一次 → 雨量 > 阈值时强制触发"避雨"子场景
- **反向验证：** 去掉雨夜压力后，被抛弃感只在对话中提到，环境不参与叙事

**映射链 5：身份焦虑（"我是谁"）→ 对话即检定 + 三轴滑条**
- **链路：** 玩家遇到同行 NPC → NPC 提问"你还在做独立游戏？" → 触发身份检定 → 玩家的信念值决定回答风格 → 回答后信念值变化 → 玩家在变化中逐步定义"我是谁"
- **Godot 实现：** `conviction` 值作为对话分支选择器 + 选择后 `conviction` 值累加 → 形成玩家身份的自定义轨迹
- **反向验证：** 去掉对话即检定后，"我是谁"变成预设好的故事而非玩家的自我探索

**映射链 6：生命倒计时（绝望）→ 3 月时钟**
- **链路：** 每个对话消耗游戏天数 → 时间逼近 3 个月 → deadline 提醒事件触发 → 玩家感知到"做不完就完了" → 绝望感积累
- **Godot 实现：** 时钟管理器追踪总天数 → 每个固定交互后消耗 1-3 天 → 总天数 >= 90 时触发终局
- **反向验证：** 去掉时钟后，绝望感少了一个核心来源——外部压力消失，只剩内心压力

**映射链 7：自我肯定 → 对话即检定 + 三轴滑条**
- **链路：** 玩家在高信念状态下次遇到知心 NPC → 触发特殊"肯定"对话分支 → NPC 给出正面反馈 → 希望 +1，信念 +0.5 → 玩家感到"我不是一个人"
- **Godot 实现：** 双条件分支：`conviction >= 6 AND hope >= 5` 时解锁"肯定"分支
- **反向验证：** 去掉三轴滑条后，自我肯定无法积累——每次都是独立事件，没有"成长感"

### 5.2 和谐度评分（AC2 — 1-5 分，附理由）

以下评分反映「主题子成分 ↔ 机制」映射对的和谐程度。评分标准：

| 分数 | 含义 |
|------|------|
| 5 | 天然绑定 — 去掉机制=去掉主题 |
| 4 | 强相关 — 机制极大地增强了主题表达 |
| 3 | 中度相关 — 机制对主题有贡献但可替代 |
| 2 | 弱相关 — 机制可能偶尔触及主题 |
| 1 | 偶发相关 — 机制基本不服务于此主题 |
| 0 | 无关 — 机制与主题没有联系 |

| 主题子成分 | 对话即检定 | 世界观滤镜 | 雨夜压力 | 3 月时钟 | 三轴滑条 |
|-----------|-----------|-----------|---------|---------|---------|
| 无力感（绝望） | 5 — 对话中"选什么都一样"的感觉直接表达无力 | 5 — 世界越冷越无力 | 4 — 雨让人停下，强化无力 | 3 — 时间压力间接增加无力 | 4 — 滑条数值下降本身就是无力感 |
| 被抛弃（绝望） | 3 — NPC 反应可强化被抛弃 | 5 — 空荡街道/冷光强化 | 5 — 雨中独行的经典意象 | 1 — 无关 | 2 — 滑条无法表达"被抛弃" |
| 自我否定（绝望） | 5 — 对话检定直接触发自我质疑 | 3 — 冷色滤镜可作为背景 | 2 — 雨夜可触发"我不行"思考 | 1 — 无关 | 5 — 信念下降=自我否定上升 |
| 他人善意（希望） | 5 — NPC 善意是被感知的核心途径 | 3 — 暖色滤镜辅助感知 | 0 — 无关 | 0 — 无关 | 2 — 希望上升可源于善意 |
| 自我肯定（希望） | 5 — 对话选择中"坚定的回答" = 自我肯定 | 1 — 极少相关 | 0 — 无关 | 0 — 无关 | 4 — 信念/希望上升 = 自我肯定 |
| 身份困惑（焦虑） | 5 — 每个对话都在追问"你是谁" | 2 — 环境可暗示迷茫 | 1 — 雨中可触发"我在哪" | 1 — 时间压力加剧困惑 | 5 — 三轴值定义你是谁 |
| 归属感（焦虑） | 4 — NPC 态度影响归属 | 3 — 灯光/环境暗示归属 | 3 — 避雨场景的"短暂归属" | 0 — 无关 | 4 — 信念值高=更有归属 |

**评分分析：**
- **最高分 28/35（对话即检定）** — 真正的核心机制，几乎承载所有主题表达
- **最低分 6/35（3 月时钟）** — 只服务于"时间压力"一个子主题，替代性好
- **中位数 17/35** — 各机制平均覆盖约一半的子主题

### 5.3 替换测试（AC3 — 至少 2 条）

**替换测试 1：将「对话即检定」替换到回合制 JRPG**

- **原上下文：** 极乐迪斯科式 CRPG，对话中隐式匹配状态值
- **替换到：** 《最终幻想》式回合制 JRPG，玩家在村庄中与 NPC 对话
- **合理性：** ✗ 不合理。JRPG 的对话系统服务于#世界观说明#和#支线任务#，玩家期待的是"选对选项获得好处"而非"选什么反映你是谁"。JRPG 的状态系统（等级/装备/技能）已经承担了人格定义的功能，对话检定与等级系统冲突。
- **结论：** 「对话即检定」与 CRPG 心理叙事类型高度绑定，不可迁移到传统 JRPG。

**替换测试 2：将「世界观滤镜」替换到《寂静岭》类心理恐怖**

- **原上下文：** 文本 CRPG，滤镜通过文字冷暖色调呈现
- **替换到：** 《寂静岭》式 3D 心理恐怖，滤镜通过画面色调/雾浓度/音效呈现
- **合理性：** ✓ 完全合理，但实现形式不同。寂静岭的表里世界切换本质上是更极致的世界观滤镜。在寂静岭中，滤镜不是文本冷暖而是视觉纹理的改变——核心设计逻辑（内心状态外化为世界呈现）完全一致。
- **结论：** 「世界观滤镜」是一个跨类型的通用设计模式，但在文本 CRPG 中以文字形式实现，在 3D 恐怖中以视觉形式实现。

**替换测试 3：将「雨夜压力」替换到《赛博朋克 2077》类开放世界**

- **原上下文：** 雨夜 = 故事的核心环境，压力 = 叙事推进器
- **替换到：** 夜之城的随机雨天，压力 = 驾驶/能见度限制
- **合理性：** △ 合理但意义不同。赛博朋克中的雨是视觉风格（氛围）而非叙事引擎。雨夜压力系统在《赛博朋克 2077》中如果作为"心情系统"的外化，需要修改整个环境系统的逻辑——当前它仅是视觉装饰。
- **结论：** 「雨夜压力」在特定游戏中可以作为"氛围机制"存在，但只有在本项目的语境下才具有"内心外化"的叙事功能。

### 5.4 消除测试（AC3 — 至少 2 条）

**消除测试 1：去掉「3 月时钟」**

| 问题 | 回答 |
|------|------|
| 去掉后主题还在吗？ | △ 部分保留 |
| 哪些主题保留？ | 绝望感（无力/被抛弃/自我否定）— 这些可以通过世界观滤镜和对话检定表达 |
| 哪些主题消失？ | "做不完就完了"的外部压力感消失 |
| 替代方案？ | 雨夜压力系统作为替代压力源（雨越大 = 压力越大） |
| 结论： | **可消除**，但会损失"时间紧迫"这一主题维度。建议在原型阶段先关闭时钟（如 PRD #5 所述），后期再按需开启。 |

**消除测试 2：去掉「世界观滤镜」**

| 问题 | 回答 |

### 5.6 Normal Path

1. 设计师查阅映射链矩阵，确定某个主题 beat 由哪个机制承载
2. 按照和谐度评分确定实现优先级（P0/P1/P2）
3. 实现者根据映射链中的"Godot 实现"描述编写代码
4. QA 使用替换/消除测试验证机制与主题的绑定强度

### 5.7 Edge Cases

1. **映射冲突：** 一个机制同时承载两个冲突主题时（例如同一个对话同时需要表达希望和绝望），优先采用"双条件分支"（如映射链 7）
2. **机制叠加：** 多个机制同时作用于同一主题时，避免过度表达（无需5个机制同时表达"无力感"）
3. **评分主观性：** 和谐度评分基于设计者判断，需在 playtest 中验证（预期评分 ±1）

### 5.8 Failure Paths

1. **实现者忽略映射链：** 直接代码实现机制但未参考映射链，导致机制与主题脱钩 → 需在 Code Review 中核对映射链
2. **映射链过时：** 后续调整机制设计时未同步更新映射链 → 建立映射链更新 check（每次机制变更时）
